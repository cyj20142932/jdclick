# 可收缩展开的悬浮菜单设计

**日期：** 2026/04/15

## 1. 方案概述

将悬浮菜单设计为两种状态：

1. **收缩状态**：只显示一个圆形图标按钮（约48dp），可拖拽移动
2. **展开状态**：显示当前完整的菜单界面

## 2. 交互设计

| 状态 | 操作 | 效果 |
|------|------|------|
| 收缩 | 点击图标 | 展开完整菜单 |
| 展开 | 点击收起按钮 | 收缩为小图标 |
| 任意 | 拖拽图标/菜单 | 移动位置 |

## 3. 布局设计

### 3.1 收缩状态

- 圆形图标按钮，直径约 48dp
- 使用统一的菜单图标（android.R.drawable.ic_menu_sort_by_size 或类似）
- 保持最小化，不遮挡屏幕
- 顶部可显示一个小绿点表示有任务在运行

### 3.2 展开状态

- 保持当前完整菜单布局不变
- 在顶部或底部添加一个"收起"按钮
- 位置记忆：展开/收起后保持上次的屏幕位置

## 4. 状态切换动画

- 展开：从圆点放大为完整菜单（约200ms，渐变+缩放）
- 收起：从完整菜单缩小为圆点（约200ms，渐变+缩放）

## 5. 持久化（可选功能）

- 使用 SharedPreferences 记住用户偏好的展开/收起状态
- 下次启动时自动恢复上次的状态

## 6. 技术实现要点

### 6.1 布局文件

- 创建 `floating_menu_collapsed.xml` 作为收缩状态的布局
- 修改 `floating_menu.xml`，添加"收起"按钮
- 使用 ViewStub 或动态切换的方式处理两种状态

### 6.2 服务逻辑修改

- 在 FloatingMenuService 中添加 `isExpanded` 状态变量
- 添加 `toggleCollapse()` 方法处理展开/收起切换
- 维护展开和收起两种状态下的位置参数

### 6.3 动画实现

- 使用 ValueAnimator 或 ObjectAnimator 实现缩放动画
- 控制 Alpha 和 Scale 属性

## 7. UI 组件清单

| 组件 | 描述 |
|------|------|
| collapsed_view | 收缩状态的小图标布局 |
| expanded_view | 展开状态的完整菜单布局 |
| btn_collapse | 收起按钮（添加到现有菜单） |
| collapse_container | 外层容器，用于切换 |
